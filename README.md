# Team Finder AD

**Веб-платформа для поиска команд и участников проектов**

Платформа позволяет пользователям создавать проекты, указывать необходимые навыки, фильтровать предложения и формировать рабочие группы.

---

## Возможности

- **Регистрация и авторизация** — вход по логину или email
- **Управление проектами** — создание, редактирование и удаление проектов
- **Управление навыками** — AJAX-подгрузка, автодополнение при вводе
- **Фильтрация проектов** — по статусу и необходимым навыкам
- **Публичные профили** — портфолио участников
- **Тестирование** — покрытие тестами (pytest) и проверка стиля (flake8)

## Быстрый старт

### 1. Скачайте все файлы

### 2. Виртуальное окружение

Перед началом работы необходимо создать и активировать виртуальное окружение Python.  


1. **Создайте виртуальное окружение (в папке проекта):**
   ```bash
   python3 -m venv venv
   ```

   После этого появится папка `venv`, где будут храниться зависимости проекта.

2. **Активируйте окружение:**

    - **Windows (PowerShell):**
      ```bash
      venv\Scripts\Activate.ps1
      ```
    - **Windows (cmd):**
      ```bash
      venv\Scripts\activate
      ```
    - **Linux/Mac:**
      ```bash
      source venv/bin/activate
      ```
---


3. **Установите зависимости из `requirements.txt`:**
   ```bash
   pip install -r requirements.txt
   ```

   После установки в окружении будут доступны все нужные библиотеки Django-проекта.
   
В файлах есть пример `.env_example`, который нужно скопировать и заполнить:

```bash
cp .env_example .env
```


### 3. Запуск проекта на локальном компьютере

Для работы проекта необходима запущенная на компьютере база данных PostgreSQL. Все данные сервера для БД хранятся в файле .env.
- Самое главное POSTGRES_PORT=5436 и TASK_VERSION=3, логин и пароль юзера =team_finder.
- После заполнения `.env` и настройки базы данных необходимо применить миграции:
```bash
python manage.py migrate
```

Теперь можно запустить сервер разработки:

```bash
python manage.py runserver
```

Теперь проект доступен по адресу [http://127.0.0.1:8000/](http://127.0.0.1:8000/).
